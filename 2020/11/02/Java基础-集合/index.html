  • 利用空参构造方法创建集合时,创建一个默认初始长度size为0的数组

  • 存满时,会扩容1.5倍
  • 如果一次添加多个元素,1.5倍还放不下,则新创建数组的长度以实际为准

3.2数据结构之数组和链表

  • 数组

    ​ 查询快、增删慢

    • 查询速度快: 查询数据通过地址值和索引定位,查询任意数据耗时相同(元素在内存中是连续存储的)
    • 删除效率低: 要将原始数据删除,同时后面每个元素前移
    • 添加效率极低: 添加位置后的每个元素后移

  • 链表

    ​ 查询慢、增删快

    • 链表中的结点是独立的对象,在内存中是不连续的,每个结点包含数据值和下一个结点的地址
    • 链表查询慢,无论查询哪个数据都要从头开始找
    • 链表增删相对快

4. Set集合

4.1 Set集合概述和特点

  • Set集合的特点
    • 无序:存取顺序不一致
    • 不重复:可以去除重复
    • 无索引:没有带索引的方法,所以不能使用普通for循环遍历,也不能通过索引来获取元素
  • Set集合的实现类
    • HashSet: 无序、不重复、无索引
    • LinkedHashSet: 有序、不重复、无索引
    • TreeSet: 可排序、不重复、无索引
  • Set集合的基本使用

4.2 HashSet的底层原理

HashSet集合底层采取哈希表存储数据

哈希表是一种增删改查数据性能都很好的结构

底层原理:

  1. 创建一个默认长度16,默认加载因子为0.75的数组,数组名为table

  2. 根据元素的哈希值和数组长度计算出应存入的位置

  3. 判断当前位置是否为null,如果是null直接存入

  4. 如果位置不为null,表示有元素,调用equals方法比较属性值

  5. 一样: 不存 不一样:存入数组 形成链表

    JDK8以前:新元素存入数组,老元素挂在新元素下面

    JDK8以后:新元素挂在老元素下面

哈希表组成

JDK8之前:数组+链表

JDK8开始:数组+链表+红黑树

4.2.1哈希值

  • 哈希值

    • 根据hashCode方法算出来的int类型的整数
    • 该方法定义在Object类中,所有对象都可以调用,默认使用地址值进行计算
    • 一般情况下,会重写hashCode方法,利用对象内部的属性值计算哈希值

  • 对象的哈希值特点

    • 如果没有重写hashCode方法,不同对象计算出的哈希值是不同的
    • 如果已经重写hashCode方法,不同对象只要属性值相同,计算出的哈希值就是一样的
    • 在小部分情况下,不同的属性值或者不同的地址值计算出来的哈希值也有可能一样(哈希碰撞)

  • 如果集合中存储的是自定义对象,必须要重写hashCode和equals方法

  • 获取哈希值的代码

4.2.2 关于HashSet的几个问题

  1. HashSet为什么存和取的顺序不一样?

    每个元素存入的时候,不能确定元素的先后存入顺序

  2. HashSet为什么没有索引

    底层是 数组+链表+红黑树存储 索引不好规定

  3. HashSet是利用什么机制保证数据去重的?

    HashCode方法 确定当前元素要存入的位置 equals方法比较对象内部的属性值是不是相同

4.3LinkedHashSet底层原理

  • 有序、不重复、无索引
  • 这里的有序指的是保证存储和取出的元素顺序一致
  • 原理: 底层数据结构依然是哈希表,每个元素又额外的多了一个双链表的机制记录存储的顺序

总结:

以后如果要数据去重,默认使用HashSet

如果要求去重且存取有序,才使用LinkedHashSet

4.4 TreeSet集合

4.4.1 TreeSet集合概述和特点

  • TreeSet集合特点

    • 不重复,无索引,可排序
    • 可排序:按照元素的默认规则**(由小到大)**排序
    • TreeSet集合底层是基于红黑树的数据结构实现排序的,增删改查性能都很好

4.4.2 TreeSet集合默认的规则:

  • 对于数值类型:Integer,Double,默认按照从小到大的顺序进行排序
  • 对于字符、字符串类型: 按照字符在ASCII码表中的数字升序进行排序

4.4.3 TreeSet的两种比较方式:

​ 方式一:默认排序/自然排序:JavaBean类实现Comparable接口指定比较规则

​ 方式二: 比较器排序: 创建TreeSet对象的时候,传递比较器Comparator指定规则

4.4.4 自然排序比较–实现Comparable接口

  • 自然排序，就是让元素所属的类实现Comparable接口，重写compareTo(T o)方法

  • 在compareTo方法中注意点:(记住)

    this:表示当前要添加的元素

    o:表示已经在红黑树存在的元素

    返回值:(用当前元素和已存在的元素作差)

    负数:认为要添加的元素是小的,存左边

    正数:认为要添加的元素是大的,存右边

    0:认为要添加的元素已经存在,舍弃

5. 数据结构

5.1 二叉树

普通的二叉树在查找的时候效率很低,因为需要遍历

5.1.1 二叉查找树(又称二叉排序树或者二叉搜索树)

特点:

​ 1.每一个节点处最多有两个子节点

​ 2.任意节点左子树上的值都小于当前节点

​ 3.任意节点右子树上的值都大于当前节点

二叉查找树添加节点

规则

​ 小的存左边

​ 大的存右边

​ 一样的不存

二叉树的遍历方式

  • 前序遍历: 根左右
  • 中序遍历(最常用):左根右
  • 后序遍历:左右根
  • 层序遍历:从根节点开始,一层一层的挨个遍历

二叉排序树弊端:

可能存在高度差过大的问题

5.1.2平衡二叉树

规则:任意节点左右子树高度差不超过1

平衡二叉树的调整:

  • 先找最小失衡子树
  • **降低高度 **
  • 保持二叉排序树的性质
  • 根据二叉排序树的性质(左子节点比根小,右子节点比根大),列出大小关系 比如 A>B>C
    此时B在中间,所以把B作为新的根节点

5.2 红黑树

  • 是一个二叉查找树
  • 高度不是平衡的
  • 每一个节点可以是红或者黑,它的平衡是通过”红黑规则“进行实现的

​ 红黑规则:

  1. 每一个节点或是红色的,或者是黑色的
  2. 根节点必须是黑色
  3. 如果一个节点没有子节点或者父节点,则该节点相应的指针属性值为Nil,这些Nil视为叶节点,每个叶节点是黑色的
    (可以理解为空节点都是Nil)
  4. 不能出现两个红色节点相连的情况(黑色节点可以相连)
  5. 对每一个节点,从该节点到其所有后代叶节点的简单路径上,均包含相同数目的黑色节点

5.2.1红黑树添加节点的规则

添加的节点默认是红色的

总结

  1. 如果想要集合中的元素可重复
    • 用ArrayList集合,基于数组的 (用的最多)
  2. 如果想要集合中的元素可重复,而且当前的增删操作明显多于查询
    • 用LinkedList集合,基于链表的
  3. 如果想要对集合中的元素去重
    • 用HashSet集合,基于哈希表的 (用的最多)
  4. 如果想对集合中的元素去重,而且保证存取顺序
    • 用LinkedHashSet,基于哈希表和双链表,效率低于HashSet
  5. 如果想对集合中的元素进行排序
    • 用TreeSet集合,基于红黑树,后续也可以用List集合实现排序

    • public static List asList(T… a)：返回由指定数组支持的固定大小的列表
    • 返回的集合不能做增删操作，可以做修改操作

  • List接口中有一个静态方法：

    • public static List of(E… elements)：返回包含任意数量元素的不可变列表
    • 返回的集合不能做增删改操作

  • Set接口中有一个静态方法：

2. HashMap

2.1 HashMap的特点

  • HashMap是Map里面的一个实现类

  • 没有其他特有方法,直接使用Map里面的方法就可以了

  • 特点都是由键决定的:无序、不重复、无索引

  • HashMap跟HashSet底层原理是一模一样的,都是哈希表结构

    底层原理:

    1. 创建一个长度为16,默认加载因子为0.75的数组

    2. 利用put方法添加数据,先创建一个entry对象,利用键计算哈希值,跟值无关

    3. 计算出要存入的索引位置,如果为null,直接添加

    4. 如果此位置之后再有元素添加进来,会调用equals方法,比较内部的属性值,如果数据是一样的,会覆盖原有的entry对象

    5. 如果数据不一样:

      ​ JDK8以前: 旧数据挂在新数据下面,形成链表

      ​ JDK8以后: 新数据挂在旧数据下面,形成链表 (长度超过8&&数组长度>=64,自动转成红黑树)

细节:

​ 依赖hashCode方法和equals方法保证键的唯一

​ 如果键存储自定义对象,需要重写hashCode方法和equals方法

​ 如果值存储自定义对象,不需要重写hashCode方法和equals方法

3.TreeMap

  • 特点

    • TreeMap跟TreeSet底层原理一样,都是红黑树结构的
    • 由键决定特性: 不重复 无索引 可排序
    • 可排序: 对键进行排序
    • 注意: 默认按照键的从小到大进行排序,也可以自己规定键的排序规则

  • package com_02.itheima_17;

import java.util.ArrayList;
import java.util.Collections;
import java.util.Random;

public class TestDemo3 {
    public static void main(String[] args) {
        /*
            班级里有5个学生
            要求:
            被点到的学生不会再被点到
            但是如果班级中的所有学生都点完了,需要重新开启第二轮点名
            注意:点名就是输出全部学生的名字
         */
        //创建学生集合
        ArrayList<String> list1=new ArrayList<>();
        //创建集合,用来存放被删除的元素
        ArrayList<String> list2=new ArrayList<>();
        //添加元素
        Collections.addAll(list1,"花木兰","李信","李白","韩信","上官婉儿","孙尚香");
        //用外循环点十轮名字
        for (int i = 1; i <= 10; i++) {
            System.out.println("-------第"+i+"轮点名开始了--------");
            //将集合长度单独定义,因为每次删除 集合的长度都会变化
            int count=list1.size();
            //随机点名(可能会有重复的)
            Random r=new Random();
            //用循环 来将集合中的名字全部随机一遍
            for (int j = 0; j < count; j++) {
                //生成随机索引
                int index = r.nextInt(list1.size());
                //为了避免重复,根据索引获取元素并且删除对应的元素
                String name = list1.remove(index);
                //将被删除的元素添加到专门的集合中
                list2.add(name);
                System.out.println(name);
            }
            //上述循环结束,表示第一轮点名结束
            //此时list1是空的  list2是满的
//        System.out.println(list1);//[]
//        System.out.println(list2);//[上官婉儿, 李白, 李信, 孙尚香, 韩信, 花木兰]
            //第一轮点名结束之后,为了保证后面几轮点名能重新开始,所以把list1的元素再添加回去,把list2置空
            list1.addAll(list2);
            list2.clear();
        }

    }
}
