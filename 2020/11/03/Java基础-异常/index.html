1.异常

1.1 异常体系介绍

  • 异常就是代表程序出现的问题

  • 不是让我们以后不出异常,而是程序出了异常之后,该如何处理

  • Java.lang.Throwable

    • Error(错误)—代表系统级别错误(属于严重问题),系统一旦出现问题,sun公司会把这些错误封装成Error对象,因此开发人员不用管
    • Exception(异常)—异常体系的最上层父类,代表程序可能出现的问题,我们通常会用Exception以及它的子类来封装程序出现的问题
      • RuntimeException(运行时异常)—RuntimeException及其子类,编译阶段不会出现异常提醒,运行时出现的异常(如:数组索引越界异常)
      • 其他异常(编译时异常)—编译阶段就会出现异常提醒(如:日期解析异常),作用在于提醒程序员

  • 代码实现

  • public class ExceptionDemo1 {
    public static void main(String[] args) throws ParseException {
        //编译时异常
        String s="2030年1月1日";
        SimpleDateFormat sdf=new SimpleDateFormat("yyyy年MM月dd日");
        //编译时,parse这里会报错,提示让我们进行修改
        Date date = sdf.parse(s);
        System.out.println(date);

        //运行时异常
        int[] arr={1,2,3,4,5};
        //编译时不会有提示,运行后报错: ArrayIndexOutOfBoundsException---数组越界异常
        System.out.println(arr[10]);

    }
}

1.3 异常的处理方式

  • 1. JVM默认的处理方式

    • 把异常的名称,异常原因以及异常出现的位置等信息输出在了控制台

    • 程序停止执行,下面的代码不会再执行了

    • package com_02.itheima_19;

public class ExceptionDemo4 {
    public static void main(String[] args) {
        System.out.println("hello");
        /*
        hello
        Exception in thread "main" java.lang.ArithmeticException: / by zero
            at com_02.itheima_19.ExceptionDemo4.main(ExceptionDemo4.java:6)
         */
        System.out.println(2/0);
        System.out.println("world");
    }
}

    • 捕获异常灵魂四问:

      • 1.如果try中没有遇到问题,怎么执行?

        • 会把try里面的代码全都执行完毕,不会执行catch里面的代码
        • 注意: 只有出现了异常才会执行catch里面的代码

      • 2.如果try中可能遇到多个问题,怎么执行?

        • 会写多个catch与之对应
        • 细节: 如果我们捕获多个异常,这些异常中如果存在父子关系的话,那么父类一定要写在下面
        • 因为exception是所有异常的顶级父类对象,如果写在上面,那不论什么异常都会执行
        • 了解: 在JDK7之后,我们可以在catch中同时捕获多个异常,中间用 | 进行隔开
          表示如果出现了A异常或B异常,采取同一种处理方案

      • 3.如果try中遇到的问题没有被捕获,怎么执行?

        • 相当于try……catch的代码白写了,此时会交给JVM虚拟机进行处理

      • 4.如果try中遇到了问题,那么try下面的其他代码还会执行吗?

        • try中下面的代码就不会执行了,直接跳转到对应的catch中,执行catch里面的语句体
        • 但是如果没有对应的catch与之匹配,那么还是会交给虚拟机进行处理

  • 3.抛出异常

    • throws

      • 注意:写在方法定义处,表示声明一个异常,告诉调用者,使用本方法可能会有哪些异常
      • 编译时异常必须要写,运行时异常可以不写

    • throw

      • 注意:写在方法内,结束方法 手动抛出异常对象,交给调用者,方法中下面的代码不再执行了
